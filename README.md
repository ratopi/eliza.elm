 # Eliza Elm

A classic [Weizenbaum ELIZA](https://en.wikipedia.org/wiki/ELIZA) chatbot implemented as a standalone, serverless web application using [Elm](https://elm-lang.org/).

## Features

- **Classic ELIZA conversation engine** ‚Äî Pattern matching and response generation based on the original Weizenbaum algorithm
- **Multi-language support** ‚Äî English and German out of the box, easily extensible to more languages
- **Personalized sessions** ‚Äî Users enter their name and select a language before chatting; Eliza addresses them by name
- **Modern UI** ‚Äî Clean, dark-themed chat interface with responsive design
- **Serverless** ‚Äî Runs entirely in the browser, no backend required
- **GitHub Pages ready** ‚Äî Automated deployment via GitHub Actions

## Getting Started

### Prerequisites

- [Elm 0.19.1](https://guide.elm-lang.org/install/elm.html)

### Build

Run the build script from the project root:

```bash
chmod +x build.sh
./build.sh
```

This compiles the Elm source code and outputs `app.js` to the `public/` directory.

### Run Locally

After building, simply open `public/index.html` in your browser. No server required!

## Adding a New Language

To add support for a new language:

1. **Create a new language module** in `elm/src/Eliza/Languages/` (e.g., `French.elm`), following the structure of `English.elm` or `German.elm`. Export: `greeting`, `goodbye`, `keywords`, `reflections`, `defaultResponses`, `quitWords`.

2. **Add a new constructor** to the `Language` type in `elm/src/Eliza/Types.elm`:
   ```elm
   type Language
       = English
       | German
       | French
   ```

3. **Wire it up** in `elm/src/Eliza/Engine.elm` ‚Äî add cases for the new language in all `case lang of` expressions.

4. **Add a language button** in `elm/src/Main.elm` in the `viewSetup` function and handle the labels.

## Deployment

The project includes a GitHub Actions workflow (`.github/workflows/deploy.yml`) that:

1. Compiles the Elm source on every push to `master`/`main`
2. Publishes the contents of `public/` to a `public` branch
3. You can then configure **GitHub Pages** to serve from the `public` branch

### Setup GitHub Pages

1. Go to your repository **Settings** ‚Üí **Pages**
2. Set **Source** to "Deploy from a branch"
3. Select the `public` branch and `/ (root)` folder
4. Save ‚Äî your site will be live!
